从前面提到的视频流程中可以了解到，交易中所有消息指令都是通过RecivedPushNotification事件获取的。

通常交易页面的消息的指令类型是OpenPage，对应的页面数据在commandParameters中。

FLOWID是交易名称，如果是新增交易，通常会为其配置一条提示

if(commandType == "OpenPage"){
    if( FLOWID == 'PersOpenAcct' ){
        if( this.state.PerlOpenAcctSTATE ){
            this.addTellerList(1,'即将为您办理',false,'个人开户')
            this.setState({
                PerlOpenAcctSTATE:false,
            })
        }
    }
}

新增交易页面主要通过BUSISTEP的值，配置在TELLERLIST中

  • type代表消息类型，交易页面通常使用座席端对话框3
  • isSure代表页面数据是否需要查询，通常根据报文中NOTICESERNO字段的值去查询
  • info代表对话框中展示的文字
  • pageName代表弹出的交易页面名称，相应路由配置在pageNode.js中
  • seeInfo代表对话框中，可以点击回跳到交易页面的文字
const TELLERLIST = {
    'ReadCustIdNo_1':{
        type:'3',
        isSure: false,
        info:'请您确认身份证信息',
        pageName:'idAuth',
        seeInfo:'查看身份证信息'
    }
}

const PageNode = [
    {
        id: 'idAuth',
        file: require('./elasticFrame/PerlOpenAcct/idAuth'),
    }
]
